# US01.7: Redesign Visual - Identidade "D2D Terminal"

**Descrição:** Reformular toda a interface do Dog Magic List para seguir uma estética "Developer to Developer" (D2D). Minimalista, baseada em tipografia monoespaçada, alta densidade de informação e componentes que remetem a consoles de comando sofisticados.

## Conceito Visual:
*   **Minimalismo Extremo:** Uso de bordas finas (1px), ausência de sombras pesadas e foco em contraste.
*   **Tipografia:** Uso obrigatório de fontes monoespaçadas (JetBrains Mono, Fira Code ou Roboto Mono).
*   **Paleta "Burned Parchment":** Fundo preto profundo (`classic-black`) com texto em creme envelhecido (`parchment-creme`), criando um visual de terminal vintage sofisticado.
*   **Interatividade:** Inputs que parecem prompts de comando (`> _`), tabelas com divisores sutis e botões em estilo "Ghost" ou "Solid Block".

## Detalhes de Alterações por Tela:

### 1. Splash Screen (System Boot):
*   Transformar o carregamento em uma sequência de boot.
*   Exibir logs rápidos: `[OK] Initializing SQLite...`, `[OK] Connecting to Scryfall...`, `[OK] Mounting UI...`.
*   Logo em ASCII Art ou versão minimalista em wireframe.

### 2. Navegação (Tabs):
*   Remover botões arredondados.
*   Usar abas estilo editor de código (VS Code tabs) ou apenas texto com prefixos numéricos `[1] Listas`, `[2] Mercado`.

### 3. Tabelas e Listas:
*   Remover cards com sombras.
*   Usar listas de linhas simples com hover highlight.
*   Preços e status alinhados à direita como colunas de dados de sistema.

### 4. Notificações:
*   Toasts que aparecem como "System Alerts" no topo ou rodapé, sem animações saltitantes, apenas fades rápidos.

## Metodologia TDI:
*   **Teste de Consistência:** Garantir que o `src/themes.ts` forneça as cores para todos os componentes.
*   **Teste de Responsividade:** Validar se a estrutura de "grid de terminal" se mantém em janelas menores.
