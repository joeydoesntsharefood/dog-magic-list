# US03.6: Budget Tuning e Controle de Edições (Deck Registry)

**Descrição:** Implementar ferramentas para otimização automática de budget (Low, Middle, High) e uma interface de seleção manual de edições para cada carta do deck após a fase de construção.

## Critérios de Aceite (AC):

### 1. Otimização de Budget (Global Tuning)
*   Adicionar botões de perfil: `[LOW_BUDGET]`, `[MID_BUDGET]`, `[HIGH_BUDGET]`.
*   **LOW:** Seleciona automaticamente a versão com o menor `priceBRL` disponível para cada carta do deck.
*   **MID:** Seleciona a versão padrão (mais recente ou média).
*   **HIGH:** Seleciona a versão com o maior `priceFoilBRL` ou arte especial (maior valor).

### 2. Nova Etapa: VERSION_FINETUNING (Step 5)
*   Após o commit inicial do deck, o usuário entra em uma tela de revisão.
*   Nesta tela, cada linha de carta exibe:
    *   **Preço Médio:** Média de valor entre todas as edições daquela carta.
    *   **Tooltip de Preços:** Ao passar o mouse no preço, listar valores de todas as versões.
    *   **Seleção Manual:** Ao clicar no preço, abrir um seletor rápido para trocar a edição daquela carta específica.

### 3. Cálculo Dinâmico
*   O custo total do deck deve ser recalculado instantaneamente ao trocar qualquer versão.
*   O status de budget (excedido ou não) deve refletir as trocas de versão.

## Estratégia Técnica:
*   **State:** Mudar `wizardDeck` para armazenar `cardVersions` (todas as versões de cada carta) permitindo a troca local sem novas chamadas de API.
*   **UI:** Componente de Tooltip/Dropdown minimalista no estilo Terminal.

## Metodologia TDI:
*   **Teste de Otimização:** Garantir que o botão `[LOW_BUDGET]` realmente selecione o menor valor numérico do array de versões.
*   **Teste de UI:** Validar que o clique no preço abre a lista de edições alternativas.
