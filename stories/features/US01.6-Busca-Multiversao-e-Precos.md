# US01.6: Busca de Múltiplas Versões e Tabela de Preços

**Descrição:** Como usuário, desejo ver todas as versões (Sets/Prints) de uma carta pesquisada para comparar imagens e preços (Foil vs Non-Foil).

## Critérios de Aceite (AC):
1.  **Busca Completa de Prints:** Ao selecionar uma carta, o backend deve retornar **todas as suas versões** (através da `prints_search_uri` do Scryfall).
2.  **Tabela de Preços:** O frontend deve exibir uma tabela simples com:
    *   Coleção (Set Name/Code).
    *   Raridade.
    *   Preço Normal (BRL).
    *   Preço Foil (BRL).
3.  **Visualização Dinâmica:** Ao clicar em uma linha da tabela de preços, a imagem exibida deve mudar para a arte correspondente àquela versão.
4.  **Consumo Direto:** As imagens não devem ser salvas localmente; devem ser carregadas via URL direta do Scryfall.
5.  **Conversão Real-Time:** Todos os valores devem ser convertidos de USD para BRL usando a cotação do momento (AwesomeAPI).

## Estratégia TDI:
*   **Teste Unitário (Backend - PriceService):** Validar a extração de múltiplos preços (Foil/Normal) de um array de versões.
*   **Teste de Integração:** Garantir que a tabela de preços seja preenchida corretamente com os valores convertidos.
