# US03.4: Importação de Deck via TXT (Bulk Import)

**Descrição:** Implementar a capacidade de colar uma lista de cartas em formato texto (ex: "1 Sol Ring") para carregar um deck inteiro no Wizard de uma só vez, processando metadados e regras automaticamente.

## Critérios de Aceite (AC):

### 1. Entrada de Dados (Step 0.5)
*   Antes ou durante o Wizard, o usuário deve ter um campo de texto (Textarea) para colar sua lista.
*   **Formato Suportado:** `[Quantidade] [Nome da Carta]` (um por linha).

### 2. Processamento de Lote (Bulk Processing)
*   O sistema deve realizar o parse de cada linha.
*   Para cada carta, deve buscar automaticamente a versão mais recente/comum no Scryfall.
*   Deve preencher: Categoria, Preço, Legalidade e Identidade de Cor.

### 3. Integração com Commander
*   Se o formato for Commander, após o parse, o sistema deve detectar se o comandante já está na lista ou solicitar que o usuário o designe.
*   Validar todo o deck importado contra a identidade de cor do comandante escolhido.

### 4. Feedback de Carregamento
*   Exibir uma barra de progresso ou logs de processamento: `[IMPORTING] Sol Ring (1/100)...`.

## Estratégia Técnica:
*   **Backend:** Novo endpoint `POST /decks/import` que recebe o texto bruto e retorna o array de `CardVersion` processado.
*   **Frontend:** Modal ou Step adicional no Wizard para colagem de texto.

## Metodologia TDI:
*   **Teste Unitário (Parser):** Validar o regex de captura de quantidade e nome.
*   **Teste de Integração:** Garantir que uma lista de 5 cartas retorne 5 objetos válidos do backend.
