# US03.2: Sistema de Validação de Regras de Formato (Legality Engine)

**Descrição:** Implementar validação automática de regras baseada no formato selecionado (Standard, Commander, Pauper). O sistema deve identificar infrações de raridade, quantidade de cópias e lista de banidos, fornecendo feedback visual sem impedir a construção.

## Regras por Formato:

### 1. COMMANDER (EDH):
*   **Legalidade:** Apenas cartas marcadas como `legal` no Scryfall para Commander.
*   **Singleton:** Máximo de 1 cópia de cada carta (exceto terrenos básicos).
*   **Deck Size:** Total de 100 cartas (avisar se for diferente).

### 2. STANDARD:
*   **Legalidade:** Apenas cartas de coleções válidas na rotação atual.
*   **Cópias:** Máximo de 4 cópias de cada carta (exceto terrenos básicos).
*   **Deck Size:** Mínimo de 60 cartas.

### 3. PAUPER:
*   **Raridade:** Apenas cartas que possuam ao menos uma impressão como **Comum**.
*   **Cópias:** Máximo de 4 cópias de cada carta (exceto terrenos básicos).
*   **Banidos:** Validar contra a lista de banidos oficial.

## Feedback de UI (D2D Style):
*   **Visual:** Cards inválidos devem possuir borda `tattoo-red`.
*   **Informativo:** Exibir Tooltip ou Label lateral com a mensagem: `[RULE_VIOLATION: TOO_MANY_COPIES]` ou `[RULE_VIOLATION: RARITY_ERROR_PAUPER]`.

## Estratégia TDI:
*   **Backend (LegalityService):** Validar funções que checam cópias e raridade.
*   **Integração:** Adicionar campos de legalidade (`legality`) no retorno da API do Scryfall.
