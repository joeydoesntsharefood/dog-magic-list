# US02.3: Inteligência de Mercado e Cache de Ofertas (LigaMagic)

**Descrição:** Evoluir o scraper para capturar preços médios e ofertas por loja, armazenando os resultados no SQLite para evitar execuções redundantes do Puppeteer.

## Critérios de Aceite (AC):
1.  **Extração Refinada:**
    *   **Preços Médios:** Capturar do container `#container-show-price` (Mínimo, Médio e Máximo nacional).
    *   **Ofertas por Loja:** Capturar do container `#container-buy-mkp` (Loja, Preço, Qtd e Link).
2.  **Sistema de Cache (SQLite):**
    *   Armazenar as ofertas no banco associadas ao `scryfall_id` ou nome único.
    *   Adicionar campo `last_updated` (timestamp).
3.  **Lógica de Atualização:**
    *   Se os dados no DB tiverem menos de 24h, carregar do banco (Instantâneo).
    *   Se forem mais antigos ou inexistentes, disparar o Puppeteer e fazer um **PATCH** no banco com os novos dados.
4.  **UI de Histórico:**
    *   Exibir no frontend: "Última atualização: DD/MM às HH:mm".
    *   Botão "Forçar Atualização" para ignorar o cache se o usuário desejar.

## Estrutura de Dados (DB):
-   `card_id`: ID Único (Scryfall).
-   `avg_price`: Valor médio nacional.
-   `offers_json`: JSON stringificado com a lista de lojas/links.
-   `updated_at`: Data da última extração.

## Estratégia TDI:
*   **Teste Unitário (DB):** Validar o PATCH de ofertas e a recuperação por tempo de expiração.
*   **Teste de Integração:** Verificar se a UI exibe a data de atualização vinda do banco.
